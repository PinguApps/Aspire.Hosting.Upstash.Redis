## Purpose
Harden the highest-risk user-facing contract that should never regress: adding this package must not ruin normal local Redis usage, and the public API should behave exactly as designed.

## Full Description
Expand the Reqnroll suite to focus specifically on the local-development contract and the final public API shape. Earlier tasks should already have added targeted tests as they introduced code; this task is the dedicated hardening pass that fills the remaining gaps and makes those core guarantees exhaustive.

The scenarios added here should cover:

- no Upstash activity when the resource is not annotated for Upstash publish behavior
- no Upstash activity during normal local execution semantics
- standard Redis references still work
- the final fluent API is discoverable and behaves consistently across overloads and modes
- management credentials remain separate from app-facing outputs

This task should be test-first where meaningful. If it finds product bugs while writing coverage, it should fix them in the same PR, but only insofar as needed to satisfy the contract under test.

## Expected Deliverables
- Comprehensive Reqnroll coverage for local no-op semantics and final API shape.
- Any focused bug fixes required to make those tests pass.

## Testing Requirements
- Add or expand scenarios for:
  - plain `AddRedis` remains unaffected
  - `AddRedis(...).PublishToUpstash(...)` still preserves local reference behavior
  - public API overloads and ownership modes are represented correctly
  - no management secret leaks into consuming-app paths
- Run the full test suite and `dotnet build ./Aspire.Hosting.Upstash.Redis.slnx`.
