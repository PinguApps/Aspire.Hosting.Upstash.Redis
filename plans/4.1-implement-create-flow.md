## Purpose
Implement the actual Upstash Redis database creation path used when ownership resolution says a new remote database must be provisioned.

## Full Description
Build the deploy-time creation flow for Upstash Redis using the typed provider client and the resolved configuration model. This task must turn a validated deployment request into a real remote database and fetch the information later tasks need for outputs.

The create flow must:

1. Build the provider create request from the package domain model.
2. Use only the supported v1 config surface:
   - database name
   - platform
   - primary region
   - read regions
   - plan
   - budget
   - eviction
   - TLS as required-on only; reject or ignore `false` according to the domain model, and do not treat TLS as mutable
3. Respect ownership resolution so it only runs when the current mode/path calls for creation.
4. Confirm the created database is ready enough for downstream reconcile and output steps.
5. Retrieve all connection details required for app-facing Redis outputs without leaking secrets in logs.

The `0.2` capability matrix requires a follow-up `GET /redis/database/{id}` without `credentials=hide` after creation. Use that response to obtain `password`, endpoint, port, TLS state, and current database settings. If `password` is missing, fail with a provider-contract error; do not call `reset-password`. Poll readiness with bounded backoff until `state == active` and `modifying_state` is empty.

This task must also define the observable deployment behavior around creation:

- what progress is reported
- what high-level success is logged
- what failure message is shown when creation fails
