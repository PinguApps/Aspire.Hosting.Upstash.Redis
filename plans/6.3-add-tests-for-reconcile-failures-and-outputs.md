Expand the Reqnroll suite to cover the full reconcile and output surface in depth. This is the hardening pass for provider-state correctness and app-facing output fidelity.

The scenarios must cover:

- no-op reconcile
- each supported mutable reconciliation path
- immutable drift failures
- unsupported mutation failures

- output generation for host, port, password, TLS, and database name
- secret redaction in diagnostics

This task should also consider whether opt-in live-provider scenarios are now justified. If yes, implement them only as explicitly opt-in tests with clear gating and documentation. If not, state why the deterministic suite remains sufficient for now. If a live-test follow-up is warranted, update `plans/` in the same PR.
