## Purpose
Expose the additional output values requested for advanced consumers without forcing every consumer to parse the Redis connection string manually.

## Full Description
Implement the extra resource outputs for:

- host / endpoint
- port
- password
- TLS enabled flag
- database name

These outputs must be populated from the credential-bearing database detail response defined in `0.2`. `password` is secret. The Upstash Management API key, REST token, read-only REST token, provider id, customer id, and billing/security metadata are not v1 application outputs.

These outputs must be designed so they are useful for advanced scenarios while still keeping the standard connection-string reference as the primary path. The output design must remain consistent with Aspire patterns and should not accidentally expose the Upstash management credentials.

Task `5.1` now generates the primary standard Redis connection string after deploy by redirecting the existing `RedisResource` connection string to the final Upstash database details. This task remains responsible for the explicit supplementary output surface for host / endpoint, port, password, TLS enabled flag, and database name beyond that primary connection string path.

This task must settle:

1. How these outputs are attached or exposed through the resource model.
2. Which outputs are treated as secrets.
3. How consuming resources can access them, if that is supported.
4. How output naming stays stable and understandable.

The implementation should avoid over-engineering. Only the agreed outputs belong in v1. Do not add speculative provider metadata just because it is available.

## Implementation Notes
Completed in task `5.2`.

- `.PublishToUpstash(...)` now attaches an `UpstashRedisOutputs` object to the existing Aspire `RedisResource`.
- Advanced consumers access the outputs with `cache.Resource.GetUpstashRedisOutputs()`.
- Stable output names are defined in `UpstashRedisOutputNames`: `Endpoint`, `Port`, `Password`, `Tls`, and `DatabaseName`.
- The deploy pipeline populates public `UpstashRedisOutputReference` instances from the credential-bearing `UpstashRedisDatabaseDetails` returned after create/adopt/reconcile succeeds.
- `Password` is the only supplementary output whose public `Secret` metadata is true; `UpstashRedisOutputs.IsSecret(...)` remains a name-based helper.
- Upstash management credentials, REST tokens, provider ids, customer ids, and billing/security metadata are not supplementary outputs.
- The standard deployed Redis connection-string output remains owned by task `5.1`.

Reqnroll coverage was added in `Features/DeployReconcileOutputs/SupplementaryOutputs.feature` for output values, stable names, secret classification, and management-key exclusion.
