Step: `6.2`

Depends on: `3.2`, `3.3`, `4.1`, `5.3`  
Can run in parallel with: `6.1`, `6.3`

## Purpose
Prove the central provisioning contract end-to-end for create-only, existing-only, and create-or-adopt behavior.

## Full Description
Expand the Reqnroll suite so ownership-mode behavior is covered as a first-class scenario matrix rather than only through unit-level behavior checks from earlier tasks. This is the hardening pass for the main deployment contract.

The test matrix should cover:

- create-only create success
- create-only existing collision failure
- existing-only success
- existing-only missing-resource failure
- create-or-adopt create path
- create-or-adopt adopt path
- repeated deploy behavior under each relevant mode
- name/identity edge cases discovered in `3.3`

Where earlier tasks already covered parts of this behavior, consolidate and deepen the scenarios instead of duplicating shallow tests.

No PR from this task is complete until the implementation, tests, `README.md`, `AGENTS.md`, and all relevant plan files describe the same current reality.
